**terraform/variables.tf:**
Add two new sensitive string variables at the end of the file:

```hcl
variable "hetzner_s3_access_key" {
  description = "Hetzner Object Storage S3 access key (from Hetzner Console)"
  type        = string
  sensitive   = true
  default     = ""
}

variable "hetzner_s3_secret_key" {
  description = "Hetzner Object Storage S3 secret key (from Hetzner Console)"
  type        = string
  sensitive   = true
  default     = ""
}
```

Default to empty string so `terraform plan` does not fail when S3 keys are not yet generated. The module call uses a `count` guard.

**terraform/main.tf:**
Add the module call at the end of the file (after the existing `module "cloudflare"` block):

```hcl
module "object_storage" {
  source = "./modules/object-storage"
  count  = var.hetzner_s3_access_key != "" ? 1 : 0

  hetzner_s3_access_key = var.hetzner_s3_access_key
  hetzner_s3_secret_key = var.hetzner_s3_secret_key
}
```

The `count` guard prevents plan failures when S3 keys are not yet created in Hetzner Console. Once the user generates keys and adds them to terraform.tfvars, the module activates.

Do NOT add `hashicorp/aws` to the root `required_providers` block. The module has its own provider declaration.
