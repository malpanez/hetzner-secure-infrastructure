2. Create NEW file `ansible/inventory/group_vars/wordpress_servers/fail2ban.yml` with fail2ban_services override that includes sshd + both nginx jails:

```yaml
---
fail2ban_services:
  - name: sshd
    enabled: true
    port: 22
    filter: sshd
    logpath: /var/log/auth.log
    maxretry: 3
    bantime: 3600
    findtime: 600
  - name: nginx-main
    enabled: true
    port: "80,443"
    filter: nginx-http-auth
    logpath: /var/log/nginx/main-access.log
    maxretry: 5
    bantime: 600
    findtime: 600
  - name: nginx-academy
    enabled: true
    port: "80,443"
    filter: nginx-http-auth
    logpath: /var/log/nginx/academy-access.log
    maxretry: 5
    bantime: 600
    findtime: 600
```

The nginx log paths come from the wordpress.conf.j2 template which renders `{{ nginx_wordpress_site_name }}-access.log`. dual-wordpress.yml sets site_name=main and site_name=academy.

The fail2ban role's jail.local.j2 loops `fail2ban_services` — this group_vars override replaces the defaults/main.yml list entirely (Ansible variable precedence: group_vars > role defaults).
