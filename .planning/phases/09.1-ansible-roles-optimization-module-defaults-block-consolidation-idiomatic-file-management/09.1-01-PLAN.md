**firewall/rules.yml** (OPT-03):

There are TWO duplicate conditional pairs to merge:

**Pair 1 — SSH allow (lines 6-27):**
- Current: "Allow SSH from management IP [relaxed idempotence]" with `changed_when: false` + `when: firewall_relax_idempotence` AND "Allow SSH from management IP" without `changed_when` + `when: not firewall_relax_idempotence`.
- Replace with SINGLE task:
  ```yaml
  - name: Firewall | Rules | Allow SSH from management IP (CRITICAL - prevents lockout)
    community.general.ufw:
      rule: allow
      port: "22"
      proto: tcp
      from_ip: "{{ item }}"
      comment: "SSH from management"
    loop: "{{ ssh_allowed_ips | default(['0.0.0.0/0']) }}"
    changed_when: not (firewall_relax_idempotence | default(false))
    tags: [firewall, rules, ssh]
  ```
  Note: `changed_when: not (firewall_relax_idempotence | default(false))` means when relax is true, changed_when is false (suppresses change); when relax is false, changed_when follows default behavior (reports changes normally). Wait — that's wrong. `changed_when: false` unconditionally suppresses. The original logic: when relax=true, always `changed_when: false`; when relax=false, default `changed_when` (i.e., report actual changes from the module). The correct merge is:
  ```yaml
  changed_when: >-
    not (firewall_relax_idempotence | default(false)) and
    _ufw_ssh_result is changed
  register: _ufw_ssh_result
  ```
  Actually simpler — `community.general.ufw` is not perfectly idempotent (always reports changed). The relaxed mode suppresses this. The correct single-task approach:
  ```yaml
  - name: Firewall | Rules | Allow SSH from management IP (CRITICAL - prevents lockout)
    community.general.ufw:
      rule: allow
      port: "22"
      proto: tcp
      from_ip: "{{ item }}"
      comment: "SSH from management"
    loop: "{{ ssh_allowed_ips | default(['0.0.0.0/0']) }}"
    changed_when: not (firewall_relax_idempotence | default(false))
    tags: [firewall, rules, ssh]
  ```
  When `firewall_relax_idempotence=true`, `changed_when` evaluates to `false` (suppresses change). When `false` or unset, `changed_when` evaluates to `true` — which means it ALWAYS reports changed. This matches the original behavior because `community.general.ufw` always reports changed anyway.

**Pair 2 — UFW ports (lines 55-76):**

- Replace with SINGLE task:

  ```yaml
  - name: Firewall | Rules | Configure UFW allowed ports

    loop: "{{ firewall_allowed_ports }}"
    changed_when: not (firewall_relax_idempotence | default(false))
    tags: [firewall, rules]
  ```

Keep all other tasks (Remove SSH rules, Deny monitoring ports, Verify SSH, Enable UFW, Verify UFW active) unchanged.
