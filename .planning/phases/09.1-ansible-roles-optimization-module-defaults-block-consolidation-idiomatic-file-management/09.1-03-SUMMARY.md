## Accomplishments

- configure.yml: 13 root:root template deploy tasks (conf.d, snippets, wordpress site, php-fpm pool) wrapped in `module_defaults` block — eliminates 39 repeated `owner: root`, `group: root`, `mode: '0644'` params
- configure.yml: 4 monitoring reverse proxy tasks wrapped in a second `module_defaults` block with shared `when: deploy_monitoring | default(false) | bool` at block level (OPT-05 bonus)
- wordpress-permissions.yml: 2 `ansible.builtin.command` find-chmod tasks replaced with single `ansible.builtin.file` with `recurse: true` and `mode: u=rwX,g=rX,o=rX` — idempotent, no shell required

## Decisions Made

- Split configure.yml into two `module_defaults` blocks by logical group rather than one mega-block — monitoring block has a shared `when` condition that also moves to block level
- `Create snippets directory` task kept outside Block A: mode is `0755` not `0644`, and it's an `ansible.builtin.file` task — adding `ansible.builtin.file` to module_defaults would risk applying owner/group/mode to `state: absent` and `state: link` tasks elsewhere
- Symbolic mode `u=rwX,g=rX,o=rX` preferred over octal: capital `X` sets execute on directories only, precisely matching the intent of separate `find -type d -chmod 755` + `find -type f -chmod 644`
