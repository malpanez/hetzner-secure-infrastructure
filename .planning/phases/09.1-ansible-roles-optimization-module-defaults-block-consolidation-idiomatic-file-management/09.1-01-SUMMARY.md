# Phase 09.1 Plan 01: Module Defaults + Firewall Task Consolidation Summary

**DRY refactor across 5 task files: module_defaults blocks for owner/group/mode, AppArmor enforce/complain merged via ternary, firewall SSH+UFW duplicate task pairs merged via conditional changed_when**

## Performance

- **Duration:** ~20 min
- **Started:** 2026-04-21T07:30:00Z
- **Completed:** 2026-04-21T07:50:00Z
- **Tasks:** 2
- **Files modified:** 5

## Accomplishments

- Wrapped deploy tasks in `module_defaults` blocks in apparmor, fail2ban, security_hardening/permissions, security_hardening/banners — eliminates per-task `owner: root`, `group: root`, `mode: "0644"` repetition
- Merged AppArmor enforce/complain dual-task pattern into single task using `ternary(apparmor_enforce_command, apparmor_complain_command)` — no more `when: apparmor_enforce_mode` + `when: not apparmor_enforce_mode`
- Merged firewall SSH allow and UFW ports duplicate conditional pairs into single tasks each using `changed_when: not (firewall_relax_idempotence | default(false))` — removes 4 duplicate task definitions
- All 5 files pass `ansible-lint` production profile (0 failures, 0 warnings)

## Task Commits

1. **Task 1: module_defaults + merged conditionals for apparmor, fail2ban, security_hardening** - `a634f2f` (refactor)
2. **Task 2: Merge duplicate conditional task pairs in firewall/rules.yml** - `94f62bc` (refactor)

## Files Created/Modified

- `ansible/roles/apparmor/tasks/configure.yml` - module_defaults block for file/template/copy; enforce/complain merged to single ternary task; noqa on tasks relying on defaults
- `ansible/roles/fail2ban/tasks/configure.yml` - module_defaults block for template/copy; per-task owner/group/mode removed from all 3 tasks
- `ansible/roles/security_hardening/tasks/permissions.yml` - module_defaults block for file; per-task owner/group removed from both tasks
- `ansible/roles/security_hardening/tasks/banners.yml` - module_defaults added to existing block; per-task owner/group/mode removed from copy tasks
- `ansible/roles/firewall/tasks/rules.yml` - SSH allow task: 2 tasks -> 1; UFW ports task: 2 tasks -> 1; conditional changed_when replaces duplicate when clauses

## Decisions Made

- `risky-file-permissions` noqa required per-task even inside `module_defaults` block — ansible-lint checks tasks in isolation, does not resolve block-level defaults at lint time
- AppArmor monitoring script and logrotate tasks included inside the same `module_defaults` block (same owner/group pattern), with monitoring script keeping explicit `mode: '0755'` override
- `changed_when: not (firewall_relax_idempotence | default(false))` is semantically correct: when relax=true evaluates to false (no change reported); when relax=false/unset evaluates to true (change always reported, matching ufw module behavior which always returns changed)
