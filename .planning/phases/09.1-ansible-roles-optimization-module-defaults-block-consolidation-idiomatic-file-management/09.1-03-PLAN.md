**wordpress-permissions.yml** (OPT-04):

Replace the two `ansible.builtin.command` tasks with `find ... chmod` with `ansible.builtin.file` using `recurse`:

Current file has 3 tasks:
1. `find {{ nginx_wordpress_web_root }} -type d -exec chmod 755 {} \;` — sets directory permissions
2. `find {{ nginx_wordpress_web_root }} -type f -exec chmod 644 {} \;` — sets file permissions
3. Secure wp-config.php — already uses `ansible.builtin.file`

Replace tasks 1 and 2 with a SINGLE `ansible.builtin.file` task:
```yaml
- name: Nginx WordPress | Permissions | Set ownership and permissions recursively
  ansible.builtin.file:
    path: "{{ nginx_wordpress_web_root }}"
    owner: "{{ nginx_wordpress_nginx_user }}"
    group: "{{ nginx_wordpress_nginx_group }}"
    mode: u=rwX,g=rX,o=rX
    recurse: true
  tags: [nginx-wordpress, wordpress, permissions]
```

The `mode: u=rwX,g=rX,o=rX` is key: capital `X` sets execute only on directories (equivalent to 755 for dirs, 644 for files). This replaces both find-chmod commands with a single idempotent task.

Keep the "Secure wp-config.php" task unchanged (it sets specific 0640 permissions and root ownership which differs from the recursive pass — it must run AFTER the recursive task).

The resulting file has exactly 2 tasks instead of 3.
