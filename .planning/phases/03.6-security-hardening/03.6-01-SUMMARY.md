# Phase 3.6 Plan 01: Fix C1 SSH cloud-init override and C4 UFW SSH anywhere rule

Fix PermitRootLogin enforcement (C1) and remove SSH allow-from-anywhere UFW rule (C4).

## What Was Done

### Task 1 — C1: SSH cloud-init override (configure.yml)

Inserted a new task in `ansible/roles/ssh_2fa/tasks/configure.yml` immediately after the `Ensure sshd_config.d directory exists` task. The task writes `/etc/ssh/sshd_config.d/00-cloud-init-hardening.conf` with `PermitRootLogin no`, overriding any cloud-init-generated drop-in that may set `PermitRootLogin yes` or `prohibit-password`.

File: `00-cloud-init-hardening.conf` uses prefix `00-` so it sorts before other drop-ins (99-hardening.conf, 50-2fa.conf) and establishes the baseline value.

### Task 2 — C4: UFW SSH allow-from-anywhere rule (rules.yml)

Added three delete tasks in `ansible/roles/firewall/tasks/rules.yml`:
1. Delete `22/tcp` rule with `from_ip: 0.0.0.0/0`
2. Delete `22/tcp` rule with `from_ip: ::/0`
3. Delete `22/tcp` rule with no `from_ip` (port-only any-source rule)

Delete tasks run after the explicit management-IP SSH allow tasks, ensuring SSH access from the management IP is established before the broad rule is removed.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Missing from_ip parameter in firewall_allowed_ports loop tasks**

- **Found during:** Task 2 — on idempotency re-run, the `22/tcp ALLOW Anywhere` rule was re-created
- **Issue:** The `Configure UFW allowed ports` tasks (both standard and relaxed-idempotence variants) iterated `firewall_allowed_ports` but did not pass `item.from_ip` to the `community.general.ufw` module. The `firewall_allowed_ports` default includes a `22/tcp` entry with `from_ip: "{{ firewall_management_ips | first }}"` — without that parameter being forwarded, UFW created an unrestricted allow rule on each run, immediately re-introducing the C4 condition.
- **Fix:** Added `from_ip: "{{ item.from_ip | default(omit) }}"` to both `Configure UFW allowed ports` task variants. Also added a third delete task (port-only, no from_ip) to clean up the pre-existing port-only rule.
- **Files modified:** `ansible/roles/firewall/tasks/rules.yml`
