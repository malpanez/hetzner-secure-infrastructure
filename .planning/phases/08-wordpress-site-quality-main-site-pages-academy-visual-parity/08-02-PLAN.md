In `ansible/roles/nginx_wordpress/tasks/wordpress-optimization.yml`, add a new section BEFORE the
"Cache Optimization" section (before line 219, the "Clear all caches" task). Insert:

```yaml
# ========================================
# Theme Cleanup (remove inactive themes)
# ========================================

- name: Nginx WordPress | Optimization | Remove inactive themes not in keep list
  ansible.builtin.shell: |
    set -euo pipefail
    KEEP="{{ nginx_wordpress_themes_keep | default(['kadence']) | join(' ') }}"
    for theme in $({{ nginx_wordpress_wpcli_path }} theme list --status=inactive --field=name --format=csv --path={{ nginx_wordpress_web_root }}); do
      if echo "$KEEP" | grep -qw "$theme"; then
        echo "KEEP: $theme"
      else
        {{ nginx_wordpress_wpcli_path }} theme delete "$theme" --path={{ nginx_wordpress_web_root }}
        echo "DELETED: $theme"
      fi
    done
  args:
    executable: /bin/bash
  become_user: "{{ nginx_wordpress_nginx_user }}"
  register: theme_cleanup
  changed_when: "'DELETED:' in theme_cleanup.stdout"
  when: wp_config_check.stat.exists
  tags: [nginx-wordpress, wordpress, optimization, themes, molecule-notest]
```

This task:
- Lists all inactive themes via WP-CLI
- Compares each against `nginx_wordpress_themes_keep` (default: [kadence])
- Deletes themes NOT in the keep list
- Is idempotent (no inactive non-kept themes = no changes)
- Runs for both sites since wordpress-optimization.yml is included for both include_role invocations
- Uses `nginx_wordpress_themes_keep` defined by 08-01 in defaults/main.yml
