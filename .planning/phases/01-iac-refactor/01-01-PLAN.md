**defaults/main.yml** (ROLE-01):
Add `nginx_wordpress_site_name: "wordpress"` after line 26 (after the `# Nginx Configuration` header, before `nginx_wordpress_server_name`). The default "wordpress" preserves backward compatibility.

Change `nginx_wordpress_php_version: "8.4"` to `nginx_wordpress_php_version: "8.3"`. The server runs PHP 8.3 (Debian 13 production). Using 8.4 would cause all socket paths to resolve to non-existent sockets, resulting in 502 errors on both sites.

```yaml
nginx_wordpress_redis_database: 0
nginx_wordpress_redis_prefix: "wp_"
```

**ROLE-04 — Parametrize all hardcoded references:**

1. **Add fastcgi_cache_path at the top of the file** (before the first `{% if nginx_wordpress_ssl_enabled %}` block, outside any `server {}` block — it is valid in `http {}` context). Insert:

```
{% if nginx_wordpress_enable_fastcgi_cache | default(true) %}
fastcgi_cache_path /var/cache/nginx/{{ nginx_wordpress_site_name }}
    levels=1:2
    keys_zone={{ nginx_wordpress_site_name }}:100m
    inactive=60m
    max_size=512m
    use_temp_path=off;
{% endif %}
```

2. **Replace all 6 fastcgi_pass occurrences** (lines 104, 140, 158, 172, 234, 259):
   - Old: `fastcgi_pass unix:/run/php/php{{ nginx_wordpress_php_version }}-fpm.sock;`
   - New: `fastcgi_pass unix:/run/php/php{{ nginx_wordpress_php_version }}-{{ nginx_wordpress_site_name }}-fpm.sock;`

3. **Replace all 4 log path references** (lines 82-83 and 205-206):
   - Old: `access_log /var/log/nginx/wordpress-access.log;` and `error_log /var/log/nginx/wordpress-error.log;`
   - New: `access_log /var/log/nginx/{{ nginx_wordpress_site_name }}-access.log;` and `error_log /var/log/nginx/{{ nginx_wordpress_site_name }}-error.log;`

4. **Replace both fastcgi_cache zone references** (lines 110 and 238):
   - Old: `fastcgi_cache wordpress;`
   - New: `fastcgi_cache {{ nginx_wordpress_site_name }};`

**ROLE-06 — Add LearnDash bypass location blocks:**

Inside the HTTPS `server {}` block, after the `# Rate-Limited Endpoints` section (after the `{% endif %}` that closes `nginx_wordpress_enable_rate_limiting`, before `# Static Asset Caching`), add:

```
{% if nginx_wordpress_learndash_enabled | default(true) %}
    location ~* ^/(ld-focus-mode|learndash-checkout)/ {
        try_files $uri $uri/ /index.php?$args;
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php{{ nginx_wordpress_php_version }}-{{ nginx_wordpress_site_name }}-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_cache_bypass 1;
        fastcgi_no_cache 1;
    }
{% endif %}
```

Note: The `wordpress-cache-bypass.conf.j2` snippet already handles LearnDash URI patterns (`/courses/`, `/lessons/`, etc.) via `$no_cache` variable — those remain untouched. ROLE-06 adds dedicated `location` blocks for `/ld-focus-mode/` and `/learndash-checkout/` which need explicit `try_files` + PHP processing + hard cache bypass.

**ROLE-07 is already handled:** The variable `nginx_wordpress_woocommerce_enabled` exists in defaults/main.yml (default: false) and the bypass snippet already has conditional WooCommerce rules. The academy invocation in the playbook (Plan 01-02) passes `nginx_wordpress_woocommerce_enabled: true` to activate them.
