<objective>
Add Redis/Valkey configuration defines to the wp-config.php.j2 template so each WordPress instance connects to a different Valkey database with a distinct key prefix.

Purpose: Prevent object cache key collisions between the main site (DB 0, prefix wp_main_) and academy site (DB 1, prefix wp_academy_). Without this, both sites would share Valkey DB 0 and their cached objects would interfere with each other.
Output: Updated wp-config.php.j2 with 5 new PHP defines driven by role variables.
</objective>

```php
define('WP_REDIS_HOST', '127.0.0.1');
define('WP_REDIS_PORT', 6379);
define('WP_REDIS_DATABASE', {{ nginx_wordpress_redis_database | default(0) }});
define('WP_REDIS_PREFIX', '{{ nginx_wordpress_redis_prefix | default("wp_") }}');
define('WP_CACHE', true);
```
