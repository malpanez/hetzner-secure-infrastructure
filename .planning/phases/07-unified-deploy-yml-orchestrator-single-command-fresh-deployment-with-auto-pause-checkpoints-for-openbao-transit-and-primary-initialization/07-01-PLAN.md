Create `ansible/playbooks/deploy.yml` with the following structure. Use FQCN for all modules, 2-space indent, no comments unless strictly necessary for operator clarity at pause points.

The playbook is a YAML list of plays and import_playbook directives in this exact sequence:

**1. Import common.yml** (base OS hardening, security, firewall)
```yaml
- name: Import common playbook
  ansible.builtin.import_playbook: common.yml
  tags: [common, security]
```

**2. Inline play: Install OpenBao (first pass, no transit token required)**
A play targeting `secrets_servers` with `become: true`, `gather_facts: true`. Apply the `openbao` role with play-level vars overriding:
- `openbao_transit_enabled: false` (skip transit token assert, write config without transit seal)
- `openbao_transit_instance_enabled: true` (still deploy transit instance)

This installs OpenBao binaries, writes non-transit config, starts transit service, and starts primary (primary starts without transit seal -- fine because it's not initialized yet).

Tag: `[openbao, install]`

**3. Inline play: Bootstrap transit instance (conditional)**
A play targeting `secrets_servers` with `become: true`, `gather_facts: false`.

Vars block (same as openbao-transit-bootstrap.yml):
```yaml
openbao_cli: "{{ openbao_binary_path | default(openbao_install_dir | default('/usr/bin') ~ '/bao') }}"
openbao_transit_api_addr: "https://127.0.0.1:{{ openbao_transit_instance_port | default(8201) }}"
openbao_env_base:
  VAULT_ADDR: "{{ openbao_transit_api_addr }}"
openbao_env: "{{ openbao_env_base | combine({'VAULT_SKIP_VERIFY': '1'}) }}"
```

Pre_tasks:
- Skip play unless `openbao_transit_bootstrap_ack | default(false) | bool` (same gate as the standalone playbook)

Tasks -- copy the EXACT task sequence from `openbao-transit-bootstrap.yml` lines 35-189, BUT:
- After the "Display transit init keys and tokens" debug task, add a conditional pause:
  ```yaml
  - name: Wait for operator to save transit credentials
    ansible.builtin.pause:
      prompt: |-
        SAVE the transit unseal keys, root token, and auto-unseal token shown above to your password manager.
        Update vault_openbao_transit_token in secrets.yml with the auto-unseal token AFTER this deployment completes.
        Press ENTER to continue the deployment.
    when:
      - transit_init is defined
      - transit_autounseal_token is defined
  ```
- After the pause, add a `set_fact` to make `deploy_transit_token` available to subsequent plays:
  ```yaml
  - name: Export transit token for subsequent plays
    ansible.builtin.set_fact:
      deploy_transit_token: "{{ transit_autounseal_token }}"
    when: transit_autounseal_token is defined
  ```
- The "Display next steps if already initialized" task at the end stays as-is.
- Do NOT include the `meta: end_play` from pre_tasks of the original -- use the assert pattern instead (same as the original playbook's pre_tasks).

Tag: `[openbao, transit, bootstrap]`

**4. Inline play: Re-deploy OpenBao with transit token (second pass)**
A play targeting `secrets_servers` with `become: true`, `gather_facts: true`.

Play-level vars:
```yaml
openbao_transit_enabled: true
openbao_transit_token: "{{ hostvars[inventory_hostname]['deploy_transit_token'] | default(vault_openbao_transit_token | default('')) }}"
```

This prefers the runtime-captured token from step 3; falls back to the vault value for re-runs where transit is already initialized.

Roles:
```yaml
- role: openbao
  tags: [openbao, secrets]
```

Pre_tasks:
- Skip this entire play if `openbao_transit_token` is empty (transit not yet bootstrapped AND no vault value):
  ```yaml
  - name: Skip OpenBao redeploy if no transit token available
    ansible.builtin.meta: end_play
    when: (hostvars[inventory_hostname]['deploy_transit_token'] | default('')) == '' and (vault_openbao_transit_token | default('')) == ''
  ```

Tag: `[openbao, secrets]`

**5. Inline play: Bootstrap primary OpenBao (conditional)**
A play targeting `secrets_servers` with `become: true`, `gather_facts: false`.

Vars block (same as openbao-bootstrap.yml):
```yaml
openbao_bootstrap_enabled: true
openbao_cli: "{{ openbao_binary_path | default('bao') }}"
openbao_env_base:
  VAULT_ADDR: "{{ openbao_api_addr }}"
openbao_env: "{{ openbao_env_base | combine(openbao_tls_self_signed | default(true) | bool | ternary({'VAULT_SKIP_VERIFY': '1'}, {})) }}"
```

Pre_tasks:
- Assert `openbao_bootstrap_ack | default(false) | bool` (same gate as standalone playbook)

Tasks:
- `include_tasks: tasks/openbao-bootstrap.yml` (this already has all the conditional logic for skipping if already initialized)

The included tasks file already displays the init keys and pauses are not built in, so add AFTER the include_tasks:

Actually, `include_tasks` runs all tasks from the file inline. The file already displays keys via debug. Add a pause AFTER the include_tasks:

```yaml
- name: Include OpenBao bootstrap tasks
  ansible.builtin.include_tasks: tasks/openbao-bootstrap.yml

- name: Wait for operator to save primary OpenBao credentials
  ansible.builtin.pause:
    prompt: |-
      SAVE the recovery keys and root token shown above to your password manager.
      Press ENTER to continue the deployment.
  when: openbao_init is defined
```

The `when: openbao_init is defined` ensures the pause only appears on fresh initialization, not on re-runs where primary is already initialized.

Tag: `[openbao, bootstrap]`

**8. Inline play: Pause for rotation token file placement (conditional)**
A play targeting `secrets_servers` with `become: true`, `gather_facts: false`.

This pause only appears on fresh initialization (when `openbao_wordpress_token is defined` — set by openbao-bootstrap.yml tasks). It asks the operator to confirm token files are in place before rotation setup runs, preventing the rotation playbook from failing with missing token files on a fresh deploy.

```yaml
- name: Wait for operator to place rotation token files
  hosts: secrets_servers
  become: true
  gather_facts: false
  tasks:
    - name: Pause for rotation token placement
      ansible.builtin.pause:
        prompt: |-
          openbao-bootstrap.yml created rotation tokens in OpenBao.
          Verify the following token files exist on the server before continuing:
            /root/.openbao-token           (wordpress rotation)
            /root/.openbao-mariadb-token   (mariadb rotation)
            /root/.openbao-academy-token   (academy rotation)
            /root/.openbao-backup-token    (backup rotation)
          They are written automatically by the bootstrap tasks. If any are missing,
          place the token manually now, then press ENTER to continue.
      when: openbao_wordpress_token is defined
```

Tag: `[openbao, rotation]`

**9. Import setup-openbao-rotation.yml** (rotation scripts and timers)

IMPORTANT design notes:
- Do NOT merge imported playbooks into deploy.yml -- they must remain independently runnable for day-2 operations.
- Monitoring runs BEFORE WordPress because the monitoring role deploys node_exporter on all hosts, and Prometheus needs to be up to start collecting.
- The `pre_tasks` with `meta: end_play` in plays 3 and 4 ensure idempotent re-runs skip already-completed steps.
- All inline plays use `hosts: secrets_servers` for OpenBao work.
- The openbao role is applied twice: first to install without transit, second to reconfigure with transit. The second pass is a no-op if transit was already configured (config.hcl unchanged = no handler triggered).
- The third pause (play 8) gates rotation setup — conditioned on `openbao_wordpress_token is defined` which is set by openbao-bootstrap.yml tasks only on fresh initialization.
